# 用户行为分析平台 · 需求文档

> 版本：v1.0 ｜ 状态：原型阶段（Demo 基于模拟数据）
> 对应原型：`用户行为分析/user_path_demo.html`

---

## 一、产品概述

面向出行业务（顺风车/打车）的用户行为分析平台，基于埋点数据接入，帮助产品、运营、数据同学定位用户在页面间的流转路径、单页面内各模块/事件的点击热力与转化，并支持版本对比与异动下钻。

平台由三个分析视角组成（顶部 Tab）：

| Tab | 名称 | 核心问题 |
| --- | --- | --- |
| ① | 页面流分析 | 用户在页面之间怎么走、在哪一步流失 |
| ② | 事件分析 | 单个页面内各模块/元素的点击热力与操作步骤 |
| ③ | 用户分析 | 分群高频路径 + 单用户行为回放 |

## 二、产品功能需求

### 2.1 通用能力（贯穿各 Tab）

- **F-0.1 查询设置**：支持「起始/终止页面或事件（正推/倒推）」、「时间范围（昨天 / 近 7 天 / 近 30 天 / 自定义）」选择。
- **F-0.2 会话间隔**：可配置数值 + 单位（分/时/天），超过该间隔则切分为新会话 / 判为流失；默认 2 小时。
- **F-0.3 统一属性筛选器**：以「添加筛选条件」方式按属性过滤，分三类（点击切换类目 + 搜索 + 选值）：
  - 事件属性：平台来源
  - 订单属性：同跨城、里程段、预约/实时单、出发时间段、品类
  - 用户属性：乘客/车主、新老客
  - 每个属性提供枚举值下拉，默认「全部」；同一属性不可重复添加。
- **F-0.4 查询触发**：配置完成后点击「查询」刷新结果。

### 2.2 页面流分析（Tab ①）

- **F-1.1 页面流转桑吉图**：以页面节点为单位展示相邻步骤的流转关系，连线宽度代表流量；每步末尾自动计算「流失」节点。
- **F-1.2 展示控制**：
  - 每步 Top：每步展示流量前 N 个节点（范围 3–10），其余归入「更多分组」。
  - 展示步数：控制展示的步骤数（范围 2–10）。
  - 桑吉图按步数自适应加宽，**超出卡片宽度时支持横向滚动**。
- **F-1.3 节点点击浮层**：点击节点弹出信息浮层，展示「页面曝光 PV / 流失量 / 流失率」，并提供：
  - 高亮节点（沿主干路径前后追踪并高亮，可取消）。
  - 查看页面详情（下钻跳转到「事件分析」，并将该页面带入为默认页面）。

### 2.3 事件分析（Tab ②）

- **F-2.1 查询配置**：与页面流分析一致（查询设置含「页面选择」+ 时间范围；筛选设置复用统一属性筛选器）。从页面流分析下钻进入时，「页面选择」默认带入上层点击的页面节点。
- **F-2.2 页面基础数据（KPI）**：PV / UV、平均停留时长、跳出率、加载时长。
- **F-2.3 页面热力图**：
  - 在真实页面截图上叠加各模块的点击区域，以颜色深浅（透明度）表达点击热度，最深不超过 80%。
  - 指标切换 Tab：点击率 / 点击占比。
  - 每个区域叠加数据气泡（白底黑字），展示模块名 + 数值。
  - 两个开关图标（黑色线性 icon）：隐藏/显示热力色块、隐藏/显示数据气泡。
- **F-2.4 版本对比**：下拉选择对比版本（如「品类中心询价页」），选中后在热力图右侧**左右并排**展示对比页截图，可关闭；空间不足时不溢出页面。
- **F-2.5 模块数据表格**：
  - 列：模块名、曝光、点击/选择、点击率、点击占比（条形可视化）、操作。
  - 「品类列表」「商品列表」提供「查看」操作。
  - 按点击率倒序排列。
  - 与热力图**双向联动**：hover 表格行 → 高亮对应热力区块（红色描边）；hover/点击热力区块 → 高亮对应表格行（含固定的模块名首列）。
  - 空间不足（如开启版本对比）时表格支持**横向滚动**，模块名首列固定。
- **F-2.6 模块操作步骤桑吉图**：
  - 默认隐藏，点击某模块后展示，**以该模块为起始节点**向后展开操作流转，节点名为模块名。
  - 控制项：起始节点（下拉，点哪个模块默认选中哪个）、会话间隔（默认 2 小时）、每步 Top（3–10）、展示步数（2–10）。
  - 样式与页面流转桑吉图一致，超长支持横向滚动。

### 2.4 用户分析（Tab ③）

- **F-3.1 分群路径检索**：指定起点页与终点页，查询中间高频路径 Top N（含各路径 UV）。
- **F-3.2 单用户路径回放**：以时间线展示某用户会话内的完整事件序列（页面曝光 / 点击 + 时间间隔）。

> 备注：用户分析 Tab 的去留待定（产品已提出可不做）。

## 三、数据需求

### 3.1 指标清单

| 指标 | 口径定义 | 应用位置 |
| --- | --- | --- |
| PV | 页面曝光次数 | 节点浮层、页面 KPI |
| UV | 页面去重访问用户数 | 页面 KPI |
| 流失量 | 进入某节点后未流向下一步的量 | 桑吉图、节点浮层 |
| 流失率 | 流失量 / 该节点 PV | 桑吉图、节点浮层 |
| 平均停留时长 | 页面停留时长均值 | 页面 KPI |
| 跳出率 | 仅访问该页即离开的占比 | 页面 KPI |
| 加载时长 | 页面加载耗时 | 页面 KPI |
| 模块曝光 | 模块在页面内的曝光量 | 模块数据表 |
| 模块点击/选择量 | 模块被点击或勾选的次数 | 模块数据表、热力图 |
| 点击率 | 模块点击量 / 模块曝光 | 模块数据表、热力图 |
| 点击占比 | 模块点击量 / 全部模块点击总量 | 模块数据表、热力图 |
| 路径 UV | 完整命中某条路径的去重用户数 | 分群路径检索 |
| 操作量 | 模块间操作流转的量 | 模块操作步骤桑吉图 |

### 3.2 维度 / 筛选属性

| 类目 | 属性 | 枚举值 |
| --- | --- | --- |
| 事件属性 | 平台来源 | Android / iOS / 微信小程序 / 支付宝 |
| 订单属性 | 同跨城 | 同城 / 跨城 |
| 订单属性 | 里程段 | 0-10km / 10-30km / 30km+ |
| 订单属性 | 预约/实时单 | 预约单 / 实时单 |
| 订单属性 | 出发时间段 | 早高峰 / 平峰 / 晚高峰 / 夜间 |
| 订单属性 | 品类 | 顺风车 / 拼车 / 打车 / 送货 |
| 用户属性 | 乘客/车主 | 乘客 / 车主 |
| 用户属性 | 新老客 | 新客 / 老客 |

- 时间维度：昨天 / 近 7 天 / 近 30 天 / 自定义区间。
- 会话维度：可配置会话间隔（分/时/天）用于切分会话与判定流失。
- 版本维度：支持页面版本对比（如不同改版方案）。

### 3.3 埋点 / 数据采集要求

- **页面级埋点**：页面曝光（含页面标识、版本号）、页面停留时长、加载时长、进入/离开事件，用于 PV/UV、停留、跳出、加载、页面流转。
- **会话与序列**：每个事件需带 用户 ID、会话 ID、时间戳、页面/事件标识、上一步来源，用于桑吉图相邻步流转、路径检索、单用户回放、按会话间隔切分。
- **模块/元素级埋点**：模块标识（需与热力图区块名一致：返回、安全中心、地图、优惠banner、高速费、品类列表、商品列表、券包、时间人数弹窗、代叫、备注、发单按钮等）、模块曝光、点击/选择事件，用于热力图与模块数据表、模块操作步骤桑吉图。
- **属性透传**：每条事件需可关联上述事件/订单/用户属性，用于筛选与下钻。
- **数据粒度与时效**：按天可查；自定义区间需支持按选定时间聚合。

### 3.4 数据校验要求

- 各节点流出（流向下一步 + 流失）应等于该节点流入/曝光，保证桑吉图守恒。
- 点击率 ∈ [0,1]；点击占比之和 = 100%。
- 模块名在「热力图区块 / 模块数据表 / 模块操作步骤桑吉图」三处需严格一致，保证联动正确。
