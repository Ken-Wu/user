# 用户路径分析工具 · 需求讨论纪要

> 记录时间：2026-06-18　|　产出物：`用户路径分析工具_PRD.md`、`user_path_demo.html`

---

## 一、背景

公司已有自建埋点平台，可通过 SQL 取页面级数据（页面曝光 PV/UV、元素点击等）。希望基于这些数据搭建一个**用户路径分析工具**，把散点埋点数据转化为可交互的路径洞察。

参考输入：一张按端（Android/iOS/Windows）拆分的页面流转桑吉图截图。

---

## 二、目标功能（三大模块）

| 模块 | 能力 | 解决的问题 |
|------|------|-----------|
| ① 流量流转桑吉图 | 可视化页面间流转，逐层下钻 | 用户从哪来、到哪去、在哪流失 |
| ② 页面功能下钻 | 单页面功能点击率 + 点击热力图 | 页面内哪些功能有效、布局是否合理 |
| ③ 用户路径回放 | 单用户/分群的真实访问序列 | 还原真实使用场景、定位异常路径 |

## 三、需求演进过程

按对话推进，记录每次调整：

1. **输出 PRD**
   - 产出 `用户路径分析工具_PRD.md`，含背景、三模块详述、数据需求、取数 SQL 思路、技术架构、里程碑、风险与 QA。

2. **生成可视化 Demo**
   - 用单文件 HTML + ECharts 实现三模块交互 demo（模拟数据），打开即可预览。

3. **界面语言改为中文**
   - 全部前端文案中文化。

4. **对照功能需求表调整模块一**
   - 新增「查询配置模块」：起始/终止页面选择（正推/倒推）、会话间隔时间、时间范围；筛选设置（事件属性·平台来源、订单属性、里程段/出发时段/品类、用户属性）；展示设置（UV/PV、topX、步数）。
   - 桑吉图规则：每步 topX + 「更多分组」聚合、每步「流失」节点、流失/更多排最下方、节点可下钻。

5. **布局：左右结构 → 上下结构**
   - 模块一查询配置从左侧栏改为顶部横向配置区，桑吉图在下方。

6. **KPI 卡片指标调整**
   - 模块一 KPI 改为漏斗四指标：询发率、发接率、接完率、发完率（带环比涨跌）。

7. **重做模块二「页面功能下钻」（对照原型图）**
   - 自上而下：筛选模块 → 页面版本 tab 切换 → 页面基础数据（PV/UV、平均停留时长、跳出率、加载时长）→ 下方左右两栏。
   - 左：页面热力图（状态切换 点击率/点击量/曝光），用手机页面线框还原打车页，叠加各元素点击率标注。
   - 右：点击排行表格（模块名/曝光/点击·选择/点击率/操作）+ 操作行为表格（操作名称/次数/操作深度）。

8. **表格与热力图联动**
   - 点击右侧点击排行表格行，左侧手机页面对应模块蓝色描边高亮并滚动到可见位置。

9. **项目迁移**
   - 将 PRD 与 demo 迁移至桌面 `用户行为分析` 文件夹。

## 四、关键设计决策

- **桑吉图取数**：会话内按时间排序打 step_no → 自连接相邻两步生成流转边 → 每步取 topX，其余并入「更多」，未进入下一步的记为「流失」。
- **流失定义**：在会话间隔时间内无后续页面访问。
- **热力图**：依赖点击坐标 `pos_x/pos_y`，按网格降采样、按屏宽归一化以适配多机型；无坐标时降级为元素维度气泡。
- **联动标识**：表格行与手机元素通过 `data-mod`（接入时对齐 element_id）实现高亮联动。
